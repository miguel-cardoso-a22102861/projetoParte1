Lista de Padrões Usados

Padrão arquitetural cliente-servidor HTTP

É um modelo de comunicação entre computadores na web. O cliente envia uma requisição para o servidor e o servidor responde com a resposta. O protocolo HTTP (Hypertext Transfer Protocol) é usado para definir a comunicação entre o cliente e o servidor. O cliente faz uma solicitação HTTP para o servidor, que envia uma resposta HTTP em retorno. Este modelo é escalável e permite que um grande número de clientes se conectem a um único servidor.

Padrão de software MVC

O Padrão de software MVC (Modelo, Visualização, Controlador) é um padrão de arquitetura de software que separa os dados da aplicação, a interface do usuário e a lógica de controlo em três componentes diferentes O Model representa os dados e a lógica de negócios do aplicativo. Ele gerencia o estado do aplicativo e fornece uma interface para acessar e manipular esses dados. A View é responsável pela apresentação dos dados do modelo para o usuário. Ela define a interface do usuário e como os dados são exibidos, por exemplo, por meio de uma interface gráfica de usuário (GUI). O Controller é responsável por receber entradas do usuário, interpretá-las e atualizar o modelo e a visão de acordo. Ele lida com a lógica de controle do aplicativo e coordena as interações entre o modelo e a visão.

Comunicação Assíncrona AJAX

AJAX significa "Asynchronous JavaScript and XML" (JavaScript e XML Assíncronos) e é um padrão de comunicação utilizado em aplicações web para enviar e receber dados do servidor sem precisar recarregar a página inteira. a comunicação entre o navegador web do usuário e o servidor ocorre em segundo plano, de forma assíncrona, sem interromper ou recarregar a página. Permite que o user tenha uma interação mais rápida, sem interrupções. O AJAX utiliza tecnologias web como HTML, CSS, JavaScript e XML ou JSON para enviar e receber dados do servidor. Quando o usuário realiza uma ação na página, como clicar em um botão, o JavaScript realizar um pedido AJAX para o servidor e recebe uma resposta em formato XML ou JSON. Em seguida, o JavaScript atualiza a página com os novos dados recebidos, sem recarregá-la.
